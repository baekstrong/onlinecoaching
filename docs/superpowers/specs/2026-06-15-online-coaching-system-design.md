# 온라인 코칭 시스템 — 설계 문서

- 작성일: 2026-06-15
- 상태: 설계 확정 (구현 계획 작성 전)

## 1. 개요 / 목적

30~40대 체력이 부족한 사람들을 대상으로, "내가 운동을 제대로 하고 있는지" 확인받고 싶어 하는
니즈를 충족하는 **비동기 영상 피드백 코칭 플랫폼**.

- 주 유입 경로: 운영 중인 유튜브 채널
- 핵심 가치: 회원이 운동 영상을 올리면, 코치가 보고 자세·방법을 교정하는 피드백을 제공
- 사업 모델: 단발성(회당) 코칭

## 2. 확정된 핵심 결정

| 항목 | 결정 |
|---|---|
| 코칭 방식 | 비동기 — 회원이 영상 업로드 → 코치가 피드백 |
| 코치 구성 | 1인 브랜드 (본인 = 유일한 코치 = 관리자) |
| 과금 모델 | 회당 결제 (단, **결제 연동은 v1 이후로 보류**) |
| 피드백 형태 | 템플릿 기반 텍스트 (이미지·영상링크 임베드 가능) |
| 영상 전달 | 플랫폼에 직접 업로드 (Cloudflare R2, 90일 후 자동삭제) |
| 앱 형태 | 모바일 웹 (반응형) |
| 로그인 | 카카오 로그인 |
| 기술 스택 | Next.js + Supabase + Cloudflare R2 |

## 3. 사용자 역할

- **회원 (Member)**: 운동하는 사람. 영상 업로드 → (추후 결제) → 피드백 열람.
- **코치 (Coach = Admin)**: 본인 1명. 요청 큐 확인 → 피드백 작성. 관리자 권한.

## 4. 핵심 사용자 흐름

### ① 회원 신청 플로우
```
유튜브 → 랜딩페이지 → 카카오 로그인 → "코칭 신청"
→ 운동 종목 선택 + 메모(궁금한 점 / 어디를 봐줬으면 하는지)
→ 영상 업로드 (R2 직접 업로드)
→ 신청 완료 (상태: 검토중)
   * v1에서는 결제 단계 없음. 코치가 수동으로 진행.
   * 결제 연동은 추후 이 단계에 삽입.
```

### ② 코치 피드백 플로우 (핵심 차별점)
```
관리자 대시보드 → 신규 요청 큐 → 영상 재생 + 회원 메모 확인
→ 피드백 작성기:
     [템플릿 불러오기] → 내용 개인화 수정 → 이미지/영상링크 삽입
→ 발행 → 회원에게 이메일 알림
```

### ③ 회원 결과 확인 플로우
```
이메일 알림 → "내 코칭함" → 피드백 열람
→ 영상은 90일 후 자동삭제, 피드백 텍스트는 영구 보관
```

## 5. 데이터 모델

### User
- id, role(`member` | `coach`), name, email, kakao_id, created_at

### CoachingRequest
- id, member_id(FK User)
- exercise_type (운동 종목)
- member_note (회원 메모)
- video_object_key (R2 키), video_uploaded_at
- status: `검토중(in_review)` → `완료(completed)` → `만료(expired)`
  - (결제 연동 시 `결제대기(pending_payment)` 상태가 앞에 추가됨)
- price (추후 결제용, v1은 null 또는 0)
- created_at

### Feedback
- id, request_id(FK CoachingRequest)
- body_rich (리치 본문: 텍스트 + 이미지 + 링크)
- published_at, created_at

### FeedbackTemplate (코치 전용)
- id, coach_id(FK User)
- title, category (예: 스쿼트 / 데드리프트 / 푸시업)
- body_rich
- created_at

### Payment (추후 구현, 스키마만 예약)
- id, request_id, amount, provider, status, paid_at

### FeedbackAsset (피드백에 첨부되는 이미지)
- id, feedback_id, image_url, created_at

## 6. 영상 처리

- 업로드: 클라이언트 압축(선택) → presigned PUT URL로 **R2에 직접 업로드** (서버 부하 0)
- 재생: R2 서명 URL
- 트랜스코딩 **없음** (MVP) — 원본 재생
- 보관 정책: **90일 라이프사이클 자동삭제** (단발성 코칭이므로 영구 보관 불필요)

### 비용 추정 (참고)
가정: 영상 1개 ≈ 100MB, 90일 보관, 건당 시청 3회.

| 규모 | 누적 저장량 | 월 전송량 | Cloudflare R2 |
|---|---|---|---|
| 초기 (50건/월) | ~15GB | ~15GB | 약 300원 |
| 성장 (300건/월) | ~90GB | ~90GB | 약 1,800원 |
| 활발 (1,000건/월) | ~300GB | ~300GB | 약 6,000원 |

R2는 전송량(egress) 무료라 영상 서비스에 압도적으로 유리. 저장소 비용은 매출의 1% 미만.

## 7. 코치 도구 (플랫폼의 핵심 무기)

비슷한 케이스가 반복되므로 코치 작업 효율이 핵심.

- **피드백 템플릿 라이브러리**: 자주 나오는 케이스(종목·문제 유형)별로 미리 작성 → 불러와 개인화
- **리치 텍스트 에디터**: 이미지 업로드 + 유튜브/영상 링크 임베드
- **요청 큐 관리**: 상태별 필터, 회신 기한 표시

## 8. 알림

- 피드백 발행 시 회원에게 **이메일 알림** (Resend)
- 카카오 알림톡은 추후

## 9. 기술 아키텍처

- **Next.js** (App Router, Vercel 배포) — 프론트 + API/서버액션 통합
- **Supabase** — Postgres DB + Auth(카카오) + Row Level Security + 이미지 Storage
- **Cloudflare R2** — 영상 저장 (presigned 업로드)
- **Resend** — 이메일 알림
- (추후) **PortOne** — 회당 결제

## 10. v1 범위 vs 추후

### v1 (지금 구현)
- 카카오 로그인
- 코칭 신청 + 영상 업로드 (결제 없이 바로 검토중)
- 코치 대시보드 + 템플릿 라이브러리 + 리치 피드백 작성/발행
- 회원 코칭함 + 피드백 열람
- 이메일 알림
- 90일 영상 자동삭제

### 추후 (YAGNI — v1 제외)
- 결제 연동 (PortOne)
- 카카오 알림톡
- 영상 위 타임스탬프 마킹/주석
- 패키지 회수권 / 구독제
- 통계 대시보드
- 네이티브 앱
- 평점/리뷰
- 영상 트랜스코딩

## 11. 비범위 (명시적 제외)

- 실시간 화상 코칭
- 다중 코치 / 코치 마켓플레이스
- 회원 간 커뮤니티 기능
